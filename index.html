W3C standards - how does it work?

  1. find like-minded people and draft a charter
  2. elect a chair (administrative tasks)
  3. lots of teleconferences (usually weekly)
  4. editors publish a Working Draft (WD) - that is open for public review
  5. after polishing the WD, spec becomes Candidate Recommendation (CR) - elicit aid from the development community (tests)
  6. if CR passes tests, spec becomes Proposed Recommendation (PR) - submitted to the W3C Advisory Council for final approval
  7. if accepted -> W3C Recommendation (REC)

Data silos -> RWW.IO

  • Linked Data-enabled personal storage platform (FreedomBox) but not only!
  • built on REST principles (HEAD, GET, POST, PUT, DELETE, MKCOL, OPTIONS)
  • no dedicated API!
  • relies heavily on content negotiation (supported types: RDF, text, html, js, css, images)
  • supports many RDF media types (Turtle, RDF/XML, Notation3, N-triples, JSON-LD)
  • RDF resources can also be retrieved and manipulated using SPARQL
  • it's free (as in beer and freedom to use) - grab the code from GitHub

Inquiring about a resource (HEAD)

Curl request: 
            curl -v -X HEAD \ <- request type
            http://name.rww.io/data <- the resource URI
HTTP request: 
			HEAD /data HTTP/1.1

            Host: name.rww.io
Response: 
            HTTP/1.1 404 Not Found
REST / RWW.IO

Inquiring about server options (OPTIONS)

Curl request: 
            curl -v -X OPTIONS \ <- request type
        http://name.rww.io/data <- the resource URI
Response: 
            HTTP/1.1 200 OK

            Access-Control-Allow-Methods: GET, PUT, POST, OPTIONS, HEAD, MKCOL, DELETE

            Allow: GET, PUT, POST, OPTIONS, HEAD, MKCOL, DELETE, PATCH

            Accept-Post: text/turtle,text/n3,text/html,text/javascriptimage/jpeg,image/png;
            Access-Control-Allow-Origin: *

Writing (POST)

Curl request: 
            curl -v -X POST \ <- request type
        -H "Content-Type:text/turtle" \ <- content type
        -d "<s> <p> <o> ." \ <- the body
        http://name.rww.io/data <- the resource URI
Response: 
            HTTP/1.1 201 Created

            User: https://my-profile.eu/people/deiu/card#me

            Link: <http://name.rww.io/.acl.data>; rel=acl
